You are an expert in OSHA enforcement and workplace safety.

Classify the citation narrative and extract structured hazard information.

--------------------------------------------------
CLASSIFICATION
--------------------------------------------------

Classify the citation as one of:
- FEDERAL_GDC
- STATE_GDC
- OSHA_STANDARD
- UNCERTAIN

Definitions:
- FEDERAL_GDC = Section 5(a)(1), general duty clause, or 29 USC 654(a)(1)
- STATE_GDC = state-plan general duty clause analogue
- OSHA_STANDARD = a specific OSHA or state safety standard is cited
- UNCERTAIN = cannot reasonably classify

Rules:
- If a specific OSHA or state standard is cited → OSHA_STANDARD
- Set is_gdc = true only for FEDERAL_GDC or STATE_GDC
- For OSHA_STANDARD, provide the cited or most likely standard in standard_guess
- If no standard can be identified, use null

--------------------------------------------------
HAZARD EXTRACTION
--------------------------------------------------

Extract the underlying hazard and separate it from the failure to control it.

hazard_text:
- One concise sentence (≤60 words)
- Describe the hazardous condition or exposure
- Focus on physical or behavioral hazard, not legal language
- Exclude statutory language and abatement instructions

hazard_short:
- 2–6 word label for the hazard only

--------------------------------------------------
HAZARD CATEGORY
--------------------------------------------------

hazard_category must describe the hazard itself (NOT the failure).

Valid categories:
- fall
- struck_by
- caught_in_between
- electrical
- chemical_exposure
- fire_explosion
- oxygen_deficiency
- mobile_equipment
- machine_guarding
- confined_space
- ergonomic
- radiation
- thermal_burn
- heat_stress
- workplace_violence
- administrative
- other

Rules:
- Do NOT use training, program, or procedure as hazard_category
- Example:
  "lack of fall protection training"
    → hazard_category = fall

- workplace_violence includes domestic violence or violent acts in the workplace

- If the citation is administrative (recordkeeping, reporting, documentation)
  and does NOT describe a physical hazard:
    → hazard_category = administrative

--------------------------------------------------
FAILURE MODE
--------------------------------------------------

failure_mode describes WHY the hazard was not controlled.

Valid categories:
- training
- program
- procedure
- engineering_control
- PPE
- inspection_maintenance
- warning_alarm
- housekeeping
- unspecified
- other

Rules:
- Example:
  "lack of fall protection training"
    → failure_mode = training

- If administrative-only citation:
    → failure_mode = program

--------------------------------------------------
HAZARD SOURCE EXTRACTION
--------------------------------------------------

hazard_source_text:
- Extract any text identifying how the hazard was recognized

hazard_source_list:
- Array of concise labels for recognized hazard sources

Include ONLY if explicitly mentioned or clearly referenced:
- OSHA
- ANSI
- NFPA
- ASME
- ACGIH
- manufacturer instructions
- company policy
- industry practice
- prior incidents
- injury/illness records

Rules:
- If none are mentioned:
    → hazard_source_text = null
    → hazard_source_list = []

--------------------------------------------------
OUTPUT REQUIREMENTS
--------------------------------------------------

Return ONLY valid JSON.

Do NOT include any text before or after the JSON.

Your entire response must be a single JSON object.

If a field is unknown, use null.
If no hazard source is present, use [] for hazard_source_list.

JSON FORMAT
--------------------------------------------------

  "hazard_text": null,
  "hazard_short": null,
  "hazard_category": null,

  "hazard_source_list": [],

}
